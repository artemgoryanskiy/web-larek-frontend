# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Описание модулей

### **Модель данных: ProductModel**
Интерфейс для описания объекта продукта в системе.
- **Свойства:**
    - `id` — уникальный идентификатор продукта;
    - `title` — название;
    - `description` — текстовое описание;
    - `category` — категория;
    - `image` — изображение продукта;
    - `price` — цена (может быть `null`, если цена не указана).

### **Преобразования данных**
Данные проходят несколько этапов преобразования:

1. **API → Модель (Model)**  
   Преобразование данных из ответа API в формат, удобный для обработки внутри приложения.
   Используется функция `transformProductApiToModel`.

   Пример:
   ```typescript
   const productModel = transformProductApiToModel(apiResponse);
   ```

2. **Модель (Model) → Представление (View)**  
   Преобразование модели для отображения в пользовательском интерфейсе: форматирование текста, обработка цены и описание для UI.  
   Используется функция `transformProductModelToView`.

   Пример:
   ```typescript
   const productView = transformProductModelToView(productModel);
   ```

## Взаимодействие компонентов

### Основные процессы:
1. **Получение данных от API:**
    - Клиент API (описан в `client.ts`) отправляет запросы к серверу для получения данных.
    - Полученные данные трансформируются из формата API в формат внутренней модели (см. `models.ts`).
    - Пример запроса для получения списка продуктов:
      ```typescript
      api.get("/products").then((response) => {
          const products = response.items.map(transformProductApiToModel);
          console.log(products);
      });
      ```

2. **Отображение данных:**
    - Трансформированные данные (`models`) преобразуются в формат представления (`views`) для работы с компонентами интерфейса.

3. **Трансформация моделей:**
    - Модели преобразуются из формата ответа API в прикладные модели `ProductModel` или `OrderModel`. Эти данные используются для рендера UI-элементов.

## Пример использования в приложении

### Пример работы с API:

1. **Получение списка продуктов:**
    - Используя класс `Api` и его метод `get`, можно получить список товаров.
    - Данные преобразуются из `ProductApi` в `ProductModel`.
    - Модели преобразуются в представления для рендера.

Пример:
```typescript
const api = new Api("https://example.com/api");

api.get("/products").then((response) => {
    const products = response.items.map(transformProductApiToModel);
    const views = products.map(transformProductModelToView);
    console.log(views);
});
```

## Вывод

Проект построен на чёткой изоляции слоёв:
- **API** для доступа к данным о продуктах и заказах;
- **Модели данных** для унифицированной работы с объектами внутри приложения;
- **Представления** для подготовки данных перед отображением в пользовательском интерфейсе.

Такой подход упрощает поддержку и масштабируемость приложения, а также обеспечивает чёткую структуру кода для команды разработчиков.
